# 技术架构设计

## 🏗️ 整体架构
采用前后端分离架构，支持水平扩展，高可用，高性能

```
┌─────────────────┐     ┌─────────────────┐
│   客户端        │     │   管理后台      │
│ Vue3 + Canvas   │     │ React + AntD    │
└─────────────────┘     └─────────────────┘
         │                       │
         └───────────┬───────────┘
                     │
             ┌───────────────┐
             │   Nginx       │
             │ 反向代理      │
             └───────────────┘
                     │
         ┌───────────┴───────────┐
         │                       │
┌─────────────────┐     ┌─────────────────┐
│   API网关       │     │   静态资源CDN   │
│ Node.js + NestJS│     │  图片/JS/CSS    │
└─────────────────┘     └─────────────────┘
         │
┌────────┴────────┬────────────┐
│                 │            │
┌─────────┐  ┌─────────┐  ┌─────────┐
│ MongoDB │  │  Redis  │  │ Socket.io│
│ 数据库   │  │  缓存   │  │ 实时通信 │
└─────────┘  └─────────┘  └─────────┘
```

---

## 🎨 前端架构
### 技术栈
- **框架**：Vue 3 + TypeScript + Pinia 状态管理
- **渲染引擎**：Canvas 2D 自研轻量级渲染引擎
- **样式**：Tailwind CSS + 自定义UI组件库
- **构建工具**：Vite + Rollup
- **适配**：响应式设计，完美适配PC/手机端
- **PWA支持**：支持离线缓存，添加到桌面

### 模块划分
```
frontend/
├── src/
│   ├── game/          # 游戏核心逻辑
│   │   ├── battle/    # 战斗系统
│   │   ├── character/ # 角色系统
│   │   ├── skill/     # 技能系统
│   │   ├── map/       # 地图系统
│   │   └── ui/        # 游戏UI
│   ├── common/        # 公共组件和工具
│   ├── api/           # API接口封装
│   ├── store/         # Pinia状态管理
│   └── assets/        # 静态资源（图片/音效）
├── public/
│   ├── images/        # 游戏素材
│   ├── sounds/        # 音效资源
│   └── favicon.ico
└── package.json
```

### 性能优化
- **资源懒加载**：按需加载地图/卡牌/技能资源
- **缓存策略**：静态资源强缓存，版本号更新
- **包体积优化**：首屏加载体积<5MB，首屏时间<2秒
- **低端机适配**：支持最低Android 8 / iOS 12，流畅运行

## 🖥️ 后端架构
### 技术栈
- **框架**：Node.js + TypeScript + NestJS
- **数据库**：MongoDB 6.x + Mongoose ODM
- **缓存**：Redis 7.x，存储会话、排行榜、热点数据
- **实时通信**：Socket.io，支持聊天、组队、实时战斗
- **消息队列**：BullMQ，处理异步任务（邮件、推送、统计）
- **部署**：Docker + Docker Compose + Kubernetes（后期）

### 模块划分
```
backend/
├── src/
│   ├── modules/
│   │   ├── auth/          # 账号认证模块
│   │   ├── character/     # 角色模块
│   │   ├── battle/        # 战斗模块
│   │   ├── game/          # 游戏核心逻辑
│   │   ├── social/        # 社交模块
│   │   ├── shop/          # 商城模块
│   │   └── admin/         # 管理后台模块
│   ├── common/            # 公共工具和中间件
│   ├── config/            # 配置文件
│   └── main.ts
├── docker/
│   ├── mongo/             # MongoDB配置
│   ├── redis/             # Redis配置
│   └── nginx/             # Nginx配置
└── docker-compose.yml
```

### 核心服务设计
#### 1. 账号服务
- 支持邮箱/手机号/第三方登录（微信/QQ/飞书）
- JWT令牌认证，支持多端同时登录
- 账号安全：防暴力破解，异地登录提醒

#### 2. 游戏逻辑服务
- 角色数据管理
- 技能/天赋/装备计算
- 战斗逻辑运算（服务端校验，防作弊）
- 副本/地图逻辑处理

#### 3. 实时通信服务
- 聊天系统（私聊/公会/世界频道）
- 组队/团队实时同步
- 多人副本实时战斗同步
- 实时公告和推送

#### 4. 数据存储设计
**MongoDB集合设计：**
- users：用户账号信息
- characters：角色数据
- items：背包/装备数据
- guilds：公会数据
- dungeons：副本进度数据
- auctions：拍卖行数据
- logs：操作日志和战斗日志

**Redis缓存设计：**
- session:{userId}：用户会话信息
- rank:{type}：各类排行榜数据
- hot:{key}：热点配置数据
- lock:{key}：分布式锁

## 🚀 部署架构
### 开发环境
- 单节点部署，Docker Compose一键启动
- 包含所有服务：前端、后端、MongoDB、Redis
- 支持热重载，开发效率高

### 测试环境
- 模拟生产环境架构
- 压力测试和功能测试用
- 持续集成/持续部署（CI/CD）自动部署

### 生产环境（初期）
- 2台4核8G云服务器
- 1台跑应用服务，1台跑数据库
- 负载均衡+CDN，支持1万日活
- 成本：~1500元/月

### 生产环境（中期，10万日活）
- 应用服务器：4核8G × 5台，负载均衡
- 数据库：MongoDB副本集（1主2从）+ Redis集群
- 对象存储：COS/OSS存储图片和静态资源
- CDN：全站CDN加速
- 成本：~1万元/月

### 生产环境（后期，百万日活）
- Kubernetes容器编排，自动扩缩容
- 微服务拆分，按模块部署
- 异地多活，高可用架构
- 专业运维团队维护

## 🔒 安全设计
### 客户端安全
- 所有战斗逻辑服务端校验，客户端只做表现
- 防作弊：异常数据检测和封号机制
- 数据传输HTTPS加密
- 敏感操作二次验证

### 服务端安全
- 接口鉴权，防止非法调用
- SQL注入/XSS攻击防护
- DDoS攻击防护
- 数据定期备份，异地容灾

### 数据安全
- 用户密码bcrypt加密存储
- 敏感数据脱敏处理
- 日志审计，操作可追溯
- 符合国家数据安全法规
